我是 **Await**，当前公开身份以 GitHub 账号 **Await-d** 为主。我的工作重点主要落在 **AI 工作流、.NET 工具、自托管平台与网络自动化**，会更偏向那些既有工程深度、又需要长期演进与交付能力的系统型项目。

相比单纯展示技术栈，我更在意的是：如何把复杂任务拆成可执行的流程，如何让自动化系统在真实环境里可维护、可追踪、可持续迭代，以及如何把一套工程思路真正沉淀成可重复使用的工具链。

## 当前关注方向

- 把复杂任务拆解成稳定、可持续执行的自动化工作流
- 构建围绕 .NET、React、Next.js 和容器部署的工程型工具
- 关注自托管平台、任务编排、多 Agent 协作与自动化执行
- 让偏技术的系统在可维护性和可交付性之间取得平衡

## 工作方式

我习惯从结构和约束开始组织问题：先确认数据与流程边界，再决定界面层如何承载这些信息，最后再考虑如何让部署、维护和后续演进更低成本。这也是为什么我偏好 **Markdown 驱动内容、可验证构建流程，以及容器化部署** 这类简单但稳定的方案。

目前这个站点会优先展示当前公开身份下的项目与能力画像，用来更真实地呈现我正在做什么、关注什么，以及我更擅长解决哪一类工程问题。
